### The "Anti-Restriction" Sideload Trick (Pure Code)

Did you know you can programmatically detect if your app was sideloaded and guide the user to fix it automatically? Android 13+ only restricts accessibility if the app doesn't know its own installer.

You can add a tiny utility function in your Jetpack Compose / Kotlin code to check for this restriction. If it's blocked, you can launch a deep link that takes the user directly to your app's specific settings page, flashing the exact "three-dot" menu they need to tap.

```kotlin
fun openSettingForRestrictedApp(context: Context) {
    try {
        // This opens FocusFlow's exact App Info system page directly
        val intent = Intent(Settings.ACTION_APPLICATION_DETAILS_SETTINGS).apply {
            data = Uri.fromParts("package", context.packageName, null)
            addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
        }
        context.startActivity(intent)
        
        // Show a quick Toast or Dialog telling them: "Tap the 3 dots in the top right and click Allow Restricted Settings!"
    } catch (e: Exception) {
        e.printStackTrace()
    }
}

```

### Why this changes everything:

1. **You own the distribution:** You just host the raw APK on GitHub or your website. No one can ban you, reject your updates, or mess up your branding name.
2. **The App fixes itself:** When a user opens FocusFlow and goes to enable the blocker, your app can detect if the system toggle is faded. It pops up a clean dialog box explaining what to do, drops them straight onto the App Info screen with one click, and they unlock it in 2 seconds.
